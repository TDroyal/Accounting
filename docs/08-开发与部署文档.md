## 1. 环境要求

### 1.1 本地开发
| 工具 | 版本 | 说明 |
|------|------|------|
| Node.js | ≥ 18 LTS | 前端 |
| pnpm | ≥ 8 | 前端包管理 |
| Go | ≥ 1.21 | 后端 |
| MySQL | 8.0 | 持久化 |
| Redis | 7.x | 缓存 |
| Docker / Compose | 最新 | 起依赖服务/部署 |

### 1.2 推荐编辑器
- VS Code + Volar（Vue）+ Go 扩展；或 GoLand + WebStorm。

## 2. 本地启动

### 2.1 起依赖（MySQL + Redis）
```bash
cd deploy
docker compose up -d mysql redis
```
`docker-compose.yml` 示例要点：
- `mysql:8.0`，端口 3306，root 密码走环境变量，挂载数据卷；
- `redis:7`，端口 6379，挂载数据卷；
- 初始化数据库 `accounting`。

### 2.2 后端
```bash
cd server
cp configs/config.example.yaml configs/config.yaml   # 填本地配置
go mod tidy
go run ./cmd/server
```
- 默认监听 `:8080`；
- 首次启动执行 migrations 建表与种子分类。

### 2.3 前端
```bash
cd web
pnpm install
pnpm dev          # 默认 http://localhost:5173
```
- `vite.config.ts` 配置 `/api` 代理到 `http://localhost:8080`。

## 3. 配置项（server/configs/config.example.yaml）

```yaml
server:
  port: 8080
  mode: debug          # debug / release

mysql:
  host: 127.0.0.1
  port: 3306
  username: root
  password: ${MYSQL_PASSWORD}
  database: accounting

redis:
  addr: 127.0.0.1:6379
  password: ${REDIS_PASSWORD}
  db: 0

jwt:
  secret: ${JWT_SECRET}
  expire: 168h         # 7 天
```

> 所有敏感值通过环境变量注入；`.env.example` 列出所需变量。

## 4. 数据库迁移

- 脚本置于 `server/migrations/`，命名 `NNN_<name>.sql`；
- 启动时由 `cmd/server` 的 init 阶段或独立命令 `go run ./cmd/migrate up` 执行；
- 种子数据：预置分类树（`001_init.sql` + `002_seed_categories.sql`）。

## 5. 构建产物

### 5.1 前端
```bash
pnpm build      # 产物 web/dist
```

### 5.2 后端
```bash
CGO_ENABLED=0 go build -o bin/server ./cmd/server
```

### 5.3 Docker 镜像
- `Dockerfile.web`：多阶段构建，nginx 托管 `dist`；
- `Dockerfile.server`：多阶段构建，最终镜像仅含二进制。

## 6. 部署架构（单机）

```
浏览器
  ↓ HTTPS
Nginx (443)
  ├─ 静态资源 → web/dist
  └─ /api/*   → 反向代理到 server:8080
  ↓
Docker Compose:
  - server (Gin)
  - mysql
  - redis
```

`deploy/docker-compose.yml` 编排上述 4 个服务，Nginx 作为前置。

## 7. CI/CD 草案（GitHub Actions）

> 单用户个人项目，CI 以「 lint + 测试 + 构建」为主，可选自动部署。

`.github/workflows/ci.yml` 要点：
1. **后端**：`go vet` + `go test ./...` + 构建；
2. **前端**：`pnpm install` + `pnpm lint` + `pnpm build`；
3. **触发**：push 到 `main` / PR；
4. **部署（可选）**：构建镜像推送 registry，SSH 到服务器 `docker compose pull && up -d`。

## 8. 测试策略

| 层 | 方式 | 工具 |
|----|------|------|
| 后端单元 | Service / Repository 单测 | Go testing + testify |
| 后端接口 | HTTP 接口测试 | httptest |
| 前端组件 | 组件测试 | Vitest + @vue/test-utils |
| E2E（可选） | 关键流程 | Playwright |

MVP 阶段优先保证**后端统计聚合**与**记账 CRUD** 的单测覆盖。

## 9. 监控与日志

- 后端：结构化日志（zap），输出到 stdout，Docker 收集；
- 关键指标：接口耗时、错误率（Prometheus 可选）；
- 前端：基础错误上报（console + 可选 Sentry）。

## 10. 开发规范

### 10.1 Git 规范
- 遵循 Conventional Commits：`feat:` / `fix:` / `docs:` / `refactor:` / `chore:` / `test:`；
- 分支：`main`（稳定）、`feat/*`、`fix/*`；
- PR 描述关联文档变更。

### 10.2 代码规范
- 前端：ESLint + Prettier，TypeScript strict；
- 后端：`gofmt` + `golangci-lint`；
- 命名：前端 camelCase，后端驼峰/帕斯卡按 Go 约定；
- 注释：复杂逻辑必注释，公共 API 用 doc 注释。

### 10.3 文档同步
- 涉及接口/表结构/架构变更时同步更新 `docs/` 对应文件；
- 文档与代码同 PR 提交。
