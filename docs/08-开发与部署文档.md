# 开发与部署文档

> 文档编号：08
> 文档版本：v1.1
> 更新日期：2026-07-12

> 变更说明（v1.1）：本地开发与运行**不使用 Docker**，MySQL / Redis 直接在本机运行（本地已安装）。删除原 Docker Compose 启动与镜像部署相关内容，改为本地直连。Docker 镜像/Compose 仅作为未来可选的生产部署方式保留在文末附录。

## 1. 环境要求

### 1.1 本地开发
| 工具 | 版本 | 说明 |
|------|------|------|
| Go | ≥ 1.24 | 后端（项目已用 go1.24 验证） |
| MySQL | 8.0 | 持久化，本地直接运行 |
| Redis | 7.x | 缓存/会话，本地直接运行 |
| Node.js | ≥ 18 LTS | 前端（后续阶段） |
| pnpm | ≥ 8 | 前端包管理（后续阶段） |

> 本机已具备 go / mysql / redis，无需 Docker。

### 1.2 推荐编辑器
- VS Code + Go 扩展；或 GoLand（后端）。前端阶段再装 Volar。

## 2. 本地启动

### 2.1 启动 MySQL 与 Redis（本机直接运行）

确认本机服务已起：
```bash
# MySQL（默认端口 3306）
mysql -uroot -e "SELECT VERSION();"

# Redis（默认端口 6379）
redis-cli ping    # 应返回 PONG
```

创建数据库：
```bash
mysql -uroot -e "CREATE DATABASE IF NOT EXISTS accounting DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

执行建表与种子分类：
```bash
mysql -uroot accounting < server/migrations/001_init.sql
mysql -uroot accounting < server/migrations/002_seed_categories.sql
```

> 若 MySQL/Redis 有密码或非默认端口，在 `config.yaml` 中调整（见 §3）。

### 2.2 后端
```bash
cd server
cp configs/config.example.yaml configs/config.yaml   # 复制并填本地配置
# 敏感字段建议用环境变量覆盖：
#   export ACCT_MYSQL_PASSWORD=...
#   export ACCT_REDIS_PASSWORD=...
#   export ACCT_JWT_SECRET=...
go mod tidy
go run ./cmd/server -c configs/config.yaml
```
- 默认监听 `:8080`；
- 健康检查：`curl http://localhost:8080/api/v1/auth/register`（应返回参数校验错误，说明服务已起）。

### 2.3 前端（后续阶段）
```bash
cd web
pnpm install
pnpm dev          # 默认 http://localhost:5173
```
- `vite.config.ts` 配置 `/api` 代理到 `http://localhost:8080`。

## 3. 配置项（server/configs/config.example.yaml）

```yaml
server:
  port: "8080"
  mode: debug          # debug / release

mysql:
  host: 127.0.0.1
  port: "3306"
  username: root
  password: ""         # 建议用环境变量 ACCT_MYSQL_PASSWORD 覆盖
  database: accounting

redis:
  addr: 127.0.0.1:6379
  password: ""         # 建议用环境变量 ACCT_REDIS_PASSWORD 覆盖
  db: 0

jwt:
  secret: ""           # 必填，建议用环境变量 ACCT_JWT_SECRET 覆盖
  expire: 168h         # 7 天

log:
  level: info          # info / warn / error
  output: stdout

limit:
  login_per_minute: 10
  transaction_per_minute: 60
```

> 所有敏感值可通过 `ACCT_*` 环境变量覆盖（前缀由 config 内 `SetEnvPrefix("ACCT")` 设定）。`config.yaml` 已被 `.gitignore` 忽略，仅提交 `config.example.yaml`。

## 4. 数据库迁移

- 脚本置于 `server/migrations/`，命名 `NNN_<name>.sql`；
- 本地用 `mysql -uroot accounting < server/migrations/NNN_xxx.sql` 手动执行；
- 种子数据：预置分类树（`001_init.sql` 建表 + `002_seed_categories.sql` 系统预置分类，注册时复制给新用户）。

## 5. 构建产物

### 5.1 后端
```bash
cd server
CGO_ENABLED=0 go build -o bin/server ./cmd/server
./bin/server -c configs/config.yaml
```

### 5.2 前端（后续阶段）
```bash
cd web
pnpm build      # 产物 web/dist
```

## 6. 部署架构（本地单机）

```
浏览器
  ↓ HTTP（本地开发） / HTTPS（生产由 Nginx 终止）
本机服务:
  - Nginx（可选，生产用）  静态资源 web/dist + 反向代理 /api → 127.0.0.1:8080
  - server (Gin)           :8080
  - MySQL                  :3306
  - Redis                  :6379
```

本地开发直接访问 `http://localhost:8080`；生产可加 Nginx 做 TLS 终止与静态托管。

## 7. CI/CD 草案（GitHub Actions）

> 单用户个人项目，CI 以「lint + 测试 + 构建」为主，可选自动部署。

`.github/workflows/ci.yml` 要点：
1. **后端**：`go vet` + `go test ./...` + `go build ./...`；
2. **前端**（后续）：`pnpm install` + `pnpm lint` + `pnpm build`；
3. **触发**：push 到 `main` / PR；
4. **部署（可选）**：构建后端二进制，SSH 到服务器重启进程（本地直连场景无需容器编排）。

## 8. 测试策略

| 层 | 方式 | 工具 |
|----|------|------|
| 后端单元 | Service / Repository 单测（内存 mock，不依赖真实 DB） | Go testing |
| 后端接口 | HTTP 接口测试 | httptest |
| 前端组件 | 组件测试 | Vitest + @vue/test-utils |
| E2E（可选） | 关键流程 | Playwright |

当前已覆盖：统计聚合、缓存失效、分类禁用校验、JWT 往返。运行：
```bash
cd server && go test ./...
```

## 9. 监控与日志

- 后端：Gin 内置 logger + Recovery，输出到 stdout；
- 关键指标：接口耗时、错误率（Prometheus 可选，后续）；
- 前端：基础错误上报（console + 可选 Sentry）。

## 10. 开发规范

### 10.1 Git 规范
- 遵循 Conventional Commits：`feat:` / `fix:` / `docs:` / `refactor:` / `chore:` / `test:`；
- 分支：`main`（稳定）、`feat/*`、`fix/*`；
- 每完成一部分核心功能即提交一次，便于回溯；
- PR/提交描述关联文档变更。

### 10.2 代码规范
- 后端：`gofmt` + `golangci-lint`；Go 驼峰/帕斯卡命名；
- 前端：ESLint + Prettier，TypeScript strict；
- 注释：每个函数加必要注释，复杂逻辑必注释，公共 API 用 doc 注释。

### 10.3 文档同步
- 涉及接口/表结构/架构变更时同步更新 `docs/` 对应文件；
- 文档与代码同提交。
