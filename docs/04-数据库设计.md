# 数据库设计

> 文档编号：04
> 文档版本：v1.0
> 更新日期：2026-07-12

---

## 1. 概述

- 数据库：MySQL 8.0，字符集 `utf8mb4`，排序规则 `utf8mb4_unicode_ci`；
- 引擎：InnoDB；
- 主键统一为 `BIGINT UNSIGNED AUTO_INCREMENT`；
- 所有业务表带 `user_id` 实现单用户体系下的数据隔离；
- 时间字段使用 `DATETIME`（业务时间）与 `TIMESTAMP`（记录时间）区分；
- 软删除：关键流水表使用 `deleted_at`，避免误删丢失数据。

## 2. ER 关系概述

```
users  1 ──── n  transactions  n ──── 1  categories
users  1 ──── n  categories (用户自定义分类)
users  1 ──── n  accounts
users  1 ──── n  budgets
transactions  n ──── 1  accounts (from_account / to_account)
```

## 3. 表结构

### 3.1 users（用户）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT UNSIGNED PK | 主键 |
| username | VARCHAR(64) | 用户名，唯一 |
| email | VARCHAR(128) | 邮箱，唯一，可空 |
| password_hash | VARCHAR(255) | bcrypt 哈希 |
| created_at | TIMESTAMP | 注册时间 |
| updated_at | TIMESTAMP | 更新时间 |

索引：`UNIQUE(username)`、`UNIQUE(email)`。

### 3.2 categories（分类树）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT UNSIGNED PK | 主键 |
| user_id | BIGINT UNSIGNED | 所属用户（系统预置分类 user_id=0） |
| parent_id | BIGINT UNSIGNED | 父分类，0 为一级 |
| name | VARCHAR(32) | 分类名 |
| type | TINYINT | 0=支出，1=转账 |
| sort | INT | 排序 |
| status | TINYINT | 0=禁用，1=启用 |
| icon | VARCHAR(64) | 图标名（可选） |
| created_at / updated_at | TIMESTAMP | —— |

索引：`INDEX(user_id, parent_id)`、`INDEX(user_id, status)`。

预置数据（系统级，user_id=0）：交通、话费、转账、外卖、吃饭、房租、购物、娱乐、其他，各自带二级示例（见 [01-需求报告](./01-需求报告.md) §4.1.3）。用户首次使用时复制一份到该用户名下，支持个性化。

### 3.3 transactions（记账流水，核心表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT UNSIGNED PK | 主键 |
| user_id | BIGINT UNSIGNED | 用户 |
| type | TINYINT | 0=支出，1=转账 |
| category_id | BIGINT UNSIGNED | 分类 |
| amount | DECIMAL(12,2) | 金额，恒为正 |
| occurred_at | DATETIME | 发生时间 |
| from_account_id | BIGINT UNSIGNED | 转出账户（可空） |
| to_account_id | BIGINT UNSIGNED | 转入账户（转账时） |
| note | VARCHAR(255) | 备注 |
| created_at / updated_at | TIMESTAMP | —— |
| deleted_at | TIMESTAMP NULL | 软删除 |

索引：
- `PRIMARY(id)`
- `INDEX(user_id, occurred_at)` —— 日/月/年统计主索引
- `INDEX(user_id, category_id)` —— 分类维度统计
- `INDEX(user_id, type, occurred_at)` —— 按类型筛选

> 金额用 `DECIMAL(12,2)` 而非浮点，避免精度问题；统计时使用 `SUM(amount)`。

### 3.4 accounts（账户，支撑转账，P2）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT UNSIGNED PK | 主键 |
| user_id | BIGINT UNSIGNED | 用户 |
| name | VARCHAR(32) | 现金/银行卡/支付宝… |
| balance | DECIMAL(12,2) | 当前余额 |
| currency | VARCHAR(8) | 币种，默认 CNY |
| sort | INT | 排序 |
| created_at / updated_at | TIMESTAMP | —— |

索引：`INDEX(user_id)`。

### 3.5 budgets（预算）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT UNSIGNED PK | 主键 |
| user_id | BIGINT UNSIGNED | 用户 |
| month | CHAR(7) | 形如 `2026-07` |
| amount | DECIMAL(12,2) | 月度预算上限 |
| created_at / updated_at | TIMESTAMP | —— |

索引：`UNIQUE(user_id, month)`。

## 4. 索引设计说明

统计查询是性能关键。所有统计都以 `(user_id, occurred_at)` 为前缀索引扫描，配合 `category_id`/`type` 二级索引。对于年度统计，`occurred_at` 范围扫描命中索引。

## 5. Redis 设计

### 5.1 会话
| Key | Value | TTL |
|-----|-------|-----|
| `session:<userId>:<jti>` | `{token meta}` | 7d |

### 5.2 统计缓存（预聚合结果）
| Key | Value | TTL | 失效时机 |
|-----|-------|-----|----------|
| `stat:d:<userId>:<YYYYMMDD>` | 当日分类聚合 JSON | 1h | 当日记账时删除 |
| `stat:m:<userId>:<YYYYMM>` | 当月聚合 JSON | 6h | 当月记账时删除 |
| `stat:y:<userId>:<YYYY>` | 当年聚合 JSON | 24h | 当年记账时删除 |

> 采用「写失效、读回源回填」的 Cache-Aside 模式，不写入时实时维护（实现简单，单用户低并发足够）。

### 5.3 限流
| Key | Value | TTL |
|-----|-------|-----|
| `rl:<ip>:<route>` | 计数 | 滑动窗口 |

### 5.4 常用分类缓存
| Key | Value | TTL |
|-----|-------|-----|
| `cats:<userId>` | 用户分类树 JSON | 1h |

## 6. 数据迁移与初始化

- 迁移脚本置于 `server/migrations/`，按版本号命名（`001_init.sql`、`002_seed_categories.sql` …）；
- 启动时由命令或 init 容器执行；
- 种子数据：预置分类树（`scripts/seed_categories.go` 或 SQL）。

## 7. 数据安全与备份

- 定期 `mysqldump` 或逻辑备份，存到独立卷/对象存储；
- 删除走软删除，30 天内可恢复；
- 敏感字段（如密码）绝不进日志、绝不导出。
