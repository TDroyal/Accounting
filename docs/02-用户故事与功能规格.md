## 2. 用户故事详情

### US-01 注册与登录
**作为**一名用户，**我希望**能注册账号并登录，**以便**只有我能访问我的记账数据。

**验收标准：**
- 注册：邮箱/用户名 + 密码，密码 bcrypt 加盐哈希存储；
- 登录成功返回 JWT，前端存于 localStorage/内存，后续请求携带；
- 会话信息存 Redis，支持登出（token 失效）；
- 未登录访问受保护接口返回 401；
- 单用户体系：注册不限制数量，但每个账号数据严格隔离。

### US-02 快速记账
**作为**一名用户，**我希望**用手机快速记一笔开销，**以便**养成记账习惯。

**验收标准：**
- 首屏即提供记账入口（浮动按钮 / 底部 Tab）；
- 录入字段：金额（必填，数字键盘）、分类（必填，常用置顶）、时间（默认当前，可改）、备注（可选）、类型（支出/转账，默认支出）；
- 单笔提交 ≤ 15s；
- 提交成功后返回新记录并刷新当日统计；
- 支持事后编辑与删除（带二次确认）。

### US-03 分类管理
**作为**一名用户，**我希望**能选择并管理支出分类，**以便**记录时快速归类、复盘时清晰分类。

**验收标准：**
- 预置分类树（交通/话费/转账/外卖/吃饭/房租/购物/娱乐/其他）；
- 一级分类下可挂二级分类（如交通→地铁/打车）；
- 用户可新增/重命名/禁用自定义分类，但不能删除已有流水的分类（改为禁用）；
- 记账时按使用频率或自定义排序展示。

### US-04 当日统计
**作为**一名用户，**我希望**查看当日支出占比与明细，**以便**知道今天花了多少、花在哪。

**验收标准：**
- 展示当日支出总额、分类占比饼图、明细列表（时间倒序）；
- 转账不计入支出总额，但可在明细中筛选查看；
- 移动端饼图可读，明细可滑动加载。

### US-05 月度统计
**作为**一名用户，**我希望**查看当月每日趋势、分类占比、与上月对比，**以便**月末复盘。

**验收标准：**
- 当月累计支出 + 当日累计；
- 每日支出折线图（横轴日期，纵轴金额）；
- 分类占比柱状图（按金额降序）；
- 与上月同期对比（环比金额与百分比）；
- 可切换任意历史月份查看。

### US-06 年度统计
**作为**一名用户，**我希望**查看年度月度趋势与 Top 分类，**以便**规划省钱节奏。

**验收标准：**
- 年度累计支出、月均支出；
- 12 个月度趋势折线/柱状；
- Top N 分类排行（可切换 N=3/5/10）；
- 可切换任意历史年份。

### US-07 预算
**作为**一名用户，**我希望**设置月度预算并看到剩余/超支提醒，**以便**控制开销。

**验收标准：**
- 可设置每月总预算（金额）；
- 统计页显示「已用 X / 预算 Y / 剩余 Z」，超支标红；
- 预算按月生效，可按月调整。

### US-08 转账
**作为**一名用户，**我希望**记录账户间转账（不计入支出），**以便**区分真实开销与资金流动。

**验收标准：**
- 转账记录：转出账户、转入账户、金额、时间、备注；
- 转账不计入支出统计总额，但出现在流水中；
- 联动账户余额（如启用账户模块）。

### US-09 数据导出
**作为**一名用户，**我希望**导出我的明细数据，**以便**本地留存与二次分析。

**验收标准：**
- 支持按时间范围导出 CSV / JSON；
- 导出字段：时间、类型、分类、金额、账户、备注；
- 文件名为 `accounting_<from>_<to>.csv`。

### US-10 响应式体验
**作为**一名用户，**我希望**在手机和电脑上都有良好体验，**以便**随时记录与复盘。

**验收标准：**
- 三档断点：<768px 手机、768–1024px 平板、≥1024px 电脑；
- 手机端：记账入口突出、数字键盘、单列明细；
- 电脑端：多列布局、图表信息密度高、支持快捷键（如 `N` 新建记账）。

## 3. 核心业务流程

### 3.1 快速记账流程
```
用户打开应用（手机）
  → 点击底部「记一笔」
  → 输入金额（数字键盘，默认聚焦）
  → 选择分类（常用置顶，最近用过的优先）
  → （可选）改时间 / 写备注
  → 提交
  → 后端校验 + 写 MySQL + 失效相关统计缓存
  → 前端刷新当日统计
```

### 3.2 月度复盘流程
```
用户在电脑端打开「统计 / 月度」
  → 选择月份（默认当月）
  → 后端读缓存 / 实时聚合
  → 前端渲染：
      · 累计 + 日趋势折线
      · 分类占比柱状
      · 环比上月对比
  → 用户判断哪些分类可压缩 → 调整下月预算
```

### 3.3 缓存失效流程
```
记账写入
  → 失效当日/当月/当年对应统计缓存键
  → 下次查询触发重新聚合并回填缓存
```

## 4. 状态定义

### 记账记录状态
| 状态 | 说明 |
|------|------|
| 支出（expense） | 计入支出统计 |
| 转账（transfer） | 资金转移，不计入支出 |

### 分类状态
| 状态 | 说明 |
|------|------|
| 启用 | 可在记账时选择 |
| 禁用 | 历史保留，记账不再展示 |

---

## 5. 边界与异常

| 场景 | 处理 |
|------|------|
| 金额非法（≤0 或非数字） | 前端拦截 + 后端校验返回 400 |
| 重复提交 | 前端按钮防抖 + 后端幂等键（备注+时间+金额哈希，可选） |
| 删除已有流水的分类 | 拒绝删除，引导禁用 |
| 统计缓存缺失 | 自动回源聚合并回填 |
| 导出数据量过大 | 后端流式写入，前端下载进度提示 |
