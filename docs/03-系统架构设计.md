## 1. 总体架构

应用采用经典三层架构，前后端分离，单体部署（单用户场景，无需微服务）。

```
┌──────────────────────────────────────────────────────┐
│  客户端（浏览器）                                       │
│  Vue 3 SPA（响应式，手机/电脑同源适配）                   │
└───────────────┬──────────────────────────────────────┘
                  │ HTTPS（Nginx 终止 TLS）
                  │ RESTful JSON
┌───────────────▼──────────────────────────────────────┐
│  后端  Gin（Go）                                       │
│  ┌─────────────────────────────────────────────────┐ │
│  │ Middleware: 日志 / Recovery / CORS / JWT 鉴权   │ │
│  ├─────────────────────────────────────────────────┤ │
│  │ Router  →  Handler(Controller)                  │ │
│  │              ↓                                   │ │
│  │           Service（业务逻辑）                      │ │
│  │              ↓                                   │ │
│  │           Repository（数据访问）                   │ │
│  └─────────────────────────────────────────────────┘ │
└──────────────┬───────────────────────┬──────────────┘
               │                       │
       ┌───────▼──────┐         ┌──────▼──────┐
       │   MySQL       │         │   Redis      │
       │ 持久化数据      │         │ 会话/缓存     │
       └───────────────┘         └──────────────┘
```

### 分层职责
| 层 | 职责 | 技术选型 |
|----|------|----------|
| 表现层 | 路由、参数校验、响应序列化 | Gin |
| 业务层 | 记账、统计、预算等业务规则 | Go service |
| 数据访问层 | ORM/SQL，隔离数据库细节 | GORM 或 sqlx |
| 缓存层 | 会话、统计预聚合、限流 | Redis |
| 持久层 | 业务数据 | MySQL 8.0 |

## 4. 请求处理流程

```
HTTP 请求
  → Nginx（静态资源直接返回；/api 代理到后端）
  → Gin Middleware 链：日志 → Recovery → CORS → JWT 鉴权
  → Router 匹配
  → Handler：参数绑定与校验
  → Service：业务逻辑（可能读写 Redis + MySQL）
  → Repository：数据库操作
  → 返回统一响应体
```

## 5. 鉴权设计

### 流程
1. 登录：校验账号密码 → 生成 JWT（access token），同时把会话写入 Redis（key: `session:<userId>:<jti>`）；
2. 请求：前端在 `Authorization: Bearer <token>` 携带 JWT；
3. 中间件校验：签名 + 过期 + Redis 会话存在；
4. 登出：删除 Redis 会话，token 立即失效。

### JWT Payload
```json
{
  "sub": "<userId>",
  "jti": "<uniqueId>",
  "exp": <unix>,
  "iat": <unix>
}
```

### 安全措施
- 密码存储：bcrypt（cost ≥ 10）；
- HTTPS 强制；
- JWT 过期时间合理（如 7 天），可配置刷新；
- 防 SQL 注入：参数化查询 / ORM；
- 限流：Redis 滑动窗口（登录接口、记账接口）。

## 6. 统一响应格式

所有接口返回统一结构：

```json
{
  "code": 0,
  "message": "ok",
  "data": { }
}
```

- `code = 0` 表示成功；非 0 为业务错误码（见 [05-API接口文档](./05-API接口文档.md)）；
- HTTP 状态码仍按语义使用（200/400/401/404/500），`code` 用于业务层区分；
- 错误响应示例：
```json
{ "code": 40001, "message": "金额必须大于 0", "data": null }
```

## 7. 错误码规范

| 区间 | 含义 |
|------|------|
| 0 | 成功 |
| 40000–40099 | 通用参数/校验错误 |
| 40100–40199 | 鉴权错误 |
| 40400–40499 | 资源不存在 |
| 50000–50099 | 服务器内部错误 |

## 8. 缓存策略概览

| 用途 | 键模式 | 失效策略 |
|------|--------|----------|
| 会话 | `session:<userId>:<jti>` | TTL 7d / 登出删除 |
| 日统计 | `stat:d:<userId>:<date>` | 记账时失效当日 |
| 月统计 | `stat:m:<userId>:<month>` | 记账时失效当月 |
| 年统计 | `stat:y:<userId>:<year>` | 记账时失效当年 |
| 限流 | `rl:<ip>:<route>` | 滑动窗口 TTL |

详见 [07-可视化与统计设计.md](./07-可视化与统计设计.md)。

## 9. 配置管理

- 配置通过环境变量 + 配置文件（`configs/config.yaml`）加载；
- 敏感信息（DB 密码、JWT 密钥）走环境变量，不入库；
- 提供 `.env.example` 作为本地开发参考。
