## 1. 设计目标

围绕「日 / 月 / 年」三种粒度，回答用户三个问题：
- **今天**：花了多少、花在哪？（占比 + 明细）
- **这个月**：每天花得稳不稳、哪类花得多、比上月如何？（趋势 + 占比 + 环比）
- **今年**：每月花得稳不稳、Top 花销是什么、月均多少？（趋势 + 排行）

图表库统一使用 **ECharts 5**，移动端与桌面端共用一套图表配置，按断点调整尺寸与图例位置。

## 2. 图表清单

| 粒度 | 图表 | 类型 | 数据维度 |
|------|------|------|----------|
| 日 | 当日分类占比 | 饼图 | 分类→金额 |
| 日 | 当日明细 | 表格 | 时间倒序 |
| 月 | 每日趋势 | 折线 | 日期→金额 |
| 月 | 分类占比 | 横向柱状 | 分类→金额（降序） |
| 月 | 环比对比 | 卡片/双柱 | 本月 vs 上月 |
| 年 | 月度趋势 | 折线/柱状 | 月→金额 |
| 年 | Top 分类 | 横向柱状 | 分类→累计金额 |

## 3. 各粒度数据结构

### 3.1 日统计响应（见 [05-API](./05-API接口文档.md) §5.1）
- `total`：当日支出总额
- `categories`：`[{category_id, category_name, amount, ratio}]`
- `transfer_total`：当日转账总额（不计入 total）

**饼图数据**：`categories.map(c => ({ name: c.category_name, value: c.amount }))`

### 3.2 月统计响应
- `total`、`prev_total`、环比 = `(total - prev_total) / prev_total`
- `trend`：`[{date, amount}]`（当月每日）
- `categories`：分类占比

**折线**：`trend` → x:date, y:amount
**柱状**：`categories` → x:category_name, y:amount

### 3.3 年统计响应
- `total`、`monthly_avg` = total / 已过月数
- `trend`：`[{month, amount}]`（12 个月）
- `top_categories`：Top N

**趋势**：`trend` → x:month, y:amount
**排行**：`top_categories` → x:category_name, y:amount

## 4. 统计 SQL（示例）

### 日统计（分类聚合）
```sql
SELECT category_id, SUM(amount) AS amount
FROM transactions
WHERE user_id = ? AND type = 0
  AND occurred_at >= ? AND occurred_at < ?   -- 当日 [00:00, 次日00:00)
  AND deleted_at IS NULL
GROUP BY category_id;
```

### 月统计（每日趋势）
```sql
SELECT DATE(occurred_at) AS date, SUM(amount) AS amount
FROM transactions
WHERE user_id = ? AND type = 0
  AND occurred_at >= ? AND occurred_at < ?   -- 当月范围
  AND deleted_at IS NULL
GROUP BY DATE(occurred_at)
ORDER BY date;
```

### 年统计（月度趋势）
```sql
SELECT DATE_FORMAT(occurred_at, '%Y-%m') AS month, SUM(amount) AS amount
FROM transactions
WHERE user_id = ? AND type = 0
  AND occurred_at >= ? AND occurred_at < ?   -- 当年范围
  AND deleted_at IS NULL
GROUP BY month
ORDER BY month;
```

> 分类名通过 `categories` 表 JOIN 获取；为减少查询，分类映射在前端用 category store 缓存。

## 5. 缓存策略（Cache-Aside）

```
读统计：
  1. 查 Redis stat:d:<userId>:<date>
  2. 命中 → 返回
  3. 未命中 → 执行 SQL 聚合 → 写回 Redis（带 TTL）→ 返回

写记账：
  1. 写 transactions
  2. 删除 stat:d/m/y 三个键（对应日/月/年）
```

### 缓存键与 TTL
| 键 | TTL |
|----|-----|
| `stat:d:<userId>:<YYYYMMDD>` | 1h |
| `stat:m:<userId>:<YYYYMM>` | 6h |
| `stat:y:<userId>:<YYYY>` | 24h |

### 一致性说明
- 单用户低并发，写失效足够保证一致性；
- 软删除/更新记账同样触发对应键删除；
- 跨日时由 TTL 兜底（过 1h 自动重建当日）。

## 6. 前端图表实现要点（composables/useECharts）

- 封装 `useECharts(domRef, option)`：挂载、resize 监听、卸载销毁；
- 按断点调整 `grid`、`legend` 位置（移动端 legend 置底）；
- 颜色：使用 dataviz 技能中的品牌中性调色板（见仓库 dataviz 资源），保证移动端可读、色盲友好；
- 空数据：显示「暂无记录」占位，不渲染空图。

## 7. 移动端适配

- 饼图：半径自适应，legend 置底横向滚动；
- 折线：减少 x 轴标签密度（隔日显示）；
- 柱状：横向展示，避免标签重叠；
- 交互：tooltip 在移动端改为点击触发。

## 8. 预算联动

- 月统计页顶部展示「已用 / 预算 / 剩余」进度条；
- `used` = 当月 `total`；预算来自 `budgets` 表（见 [04-数据库设计](./04-数据库设计.md)）；
- 超支时进度条变红 + 提示文案。
